# Agents Catalog

Agents are grouped by functional domain to keep navigation simple.

## Layout

- `intake/`
  - `intake_agent`

- `design/`
  - `ui_research_agent`
  - `design_brief_agent`
  - `ui_design_agent`
  - `ui_ux_spec_agent`

- `architecture/`
  - `solution_arch_agent`
  - `api_contract_agent`

- `implementation/`
  - `frontend_dev_agent`
  - `backend_dev_agent`
  - `db_schema_agent`

- `testing/`
  - `test_planner_agent`
  - `unit_test_agent`
  - `integration_test_agent`
  - `e2e_test_agent`

- `quality/`
  - `qa_gate_agent`

- `operations/`
  - `bugfix_agent`

- `release/`
  - `release_agent`
  - `post_deploy_verify_agent`

Each agent folder contains an `AGENT.md` file with role, I/O, stage responsibilities, and handoff contract.
