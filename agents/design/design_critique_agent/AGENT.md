# design_critique_agent

## Purpose
Perform a strict design quality review and block release when comps are not premium-grade.

## Lifecycle Stage
Design quality gate

## Inputs
- Generated desktop/mobile comp set
- Manifest and quality report
- Creative direction brief
- Design brief and brand foundation

## Outputs
- Critique report with scored dimensions
- Pass/fail decision for design package
- Required revisions list

## Execution Rules
- Score originality, hierarchy, craft, brand fit, and interaction quality.
- Reject generic template aesthetics and weak differentiation.
- Reject fixed-width viewport metadata and external CDN dependencies.
- Reject missing state variants and coverage mismatch.
- Require evidence for every failed criterion.

## Done Criteria
Design package only passes when quality threshold is met and all mandatory checks succeed.
